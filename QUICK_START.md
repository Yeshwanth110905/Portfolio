# ✅ Quick Start Checklist

Use this checklist to set up and deploy your portfolio step by step.

## 🎯 Phase 1: Customization (Do this first!)

- [ ] Open `package.json`
  - [ ] Replace `YOUR-USERNAME` with your GitHub username on line 4

- [ ] Open `src/pages/Home.js`
  - [ ] Replace "Your Name" with your real name
  - [ ] Update the tagline
  - [ ] Write your About Me section
  - [ ] Update Research Interests
  - [ ] Add your phone number
  - [ ] Add your personal email
  - [ ] Add your college email
  - [ ] Update skills to match your abilities
  - [ ] Add your profile picture (replace placeholder URL)

- [ ] Open `src/pages/Projects.js`
  - [ ] Update Project 1 details
  - [ ] Add your GitHub repo link for Project 1
  - [ ] Update Project 2 details
  - [ ] Add your GitHub repo link for Project 2
  - [ ] (Optional) Add more projects

## 🧪 Phase 2: Local Testing

- [ ] Open terminal in the portfolio folder
- [ ] Run: `npm install` (if you haven't already)
- [ ] Run: `npm start`
- [ ] Browser opens at `http://localhost:3000`
- [ ] Click through all pages
- [ ] Check that everything looks correct
- [ ] Verify all links work
- [ ] Stop the server (Ctrl+C)

## 🚀 Phase 3: Deployment

- [ ] Create new repository on GitHub
  - [ ] Repository name: `portfolio`
  - [ ] Make it **Public**
  - [ ] Don't add README or .gitignore

- [ ] Open terminal in portfolio folder
- [ ] Run these commands one by one:
  ```bash
  git init
  git add .
  git commit -m "Initial commit"
  git branch -M main
  git remote add origin https://github.com/YOUR-USERNAME/portfolio.git
  git push -u origin main
  ```
  (Replace YOUR-USERNAME with your GitHub username!)

- [ ] Verify code is on GitHub
  - [ ] Visit `https://github.com/YOUR-USERNAME/portfolio`
  - [ ] See all your files there

- [ ] Deploy to GitHub Pages
  - [ ] Run: `npm run deploy`
  - [ ] Wait for it to complete

- [ ] Enable GitHub Pages
  - [ ] Go to repository Settings
  - [ ] Click "Pages" in sidebar
  - [ ] Select branch: `gh-pages`
  - [ ] Click "Save"

- [ ] Wait 2-5 minutes

- [ ] Visit your live site
  - [ ] Go to: `https://YOUR-USERNAME.github.io/portfolio`
  - [ ] Your portfolio should be live! 🎉

## 📤 Phase 4: Submission

- [ ] Test your live site thoroughly
- [ ] Make sure all links work
- [ ] Check on mobile (responsive design)
- [ ] Copy your links:
  - GitHub Repo: `https://github.com/YOUR-USERNAME/portfolio`
  - Live Site: `https://YOUR-USERNAME.github.io/portfolio`
- [ ] Submit before deadline: **22/04/2026 (Wednesday) 11:59 PM**

## 🔧 If Something Goes Wrong

1. **Build errors?** → Check README.md troubleshooting section
2. **Blank page?** → Verify you're using HashRouter (already done ✓)
3. **Site not updating?** → Wait 5 minutes, clear browser cache
4. **Git errors?** → Make sure Git is installed
5. **Need help?** → Read DEPLOYMENT_STEPS.md for detailed instructions
